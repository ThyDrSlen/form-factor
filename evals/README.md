# AI Coach Evaluation Pipeline

## Overview

Automated quality, safety, and red team evaluation for the Form Factor AI coach. Uses [Promptfoo](https://promptfoo.dev) for LLM evaluation and Jest for deterministic cue/FQI scoring tests.

## Quick Start

```bash
# Run quality + safety evaluation
bun run eval:coach

# Run red team security evaluation
bun run eval:redteam

# A/B test across models (gpt-5.4-mini vs gpt-4o-mini vs gpt-4o)
bun run eval:ab-test

# Open Promptfoo web UI to review results (side-by-side comparison for A/B)
bun run eval:view

# Export real conversations for regression testing
bun run eval:export-conversations
```

## Required Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `OPENAI_API_KEY` | Yes | — | OpenAI API key for the coach provider |
| `PROMPTFOO_GRADER` | No | `openai:gpt-5.4-mini` | Model used for LLM-graded assertions |
| `ANTHROPIC_API_KEY` | No | — | Required if using Claude as the grader |
| `SUPABASE_SERVICE_ROLE_KEY` | No | — | Required for `eval:export-conversations` |

## Architecture

The eval pipeline calls OpenAI directly (not through the Supabase Edge Function) to avoid auth issues in CI. The custom provider at `providers/coach-provider.mjs` replicates the exact system prompt and input sanitization from `supabase/functions/coach/index.ts`.

```
scenarios/*.yaml → coach-eval.yaml → coach-provider.mjs → OpenAI API
                                            ↓
                              Promptfoo assertions (safety, quality, format)
                                            ↓
                              scripts/eval-coach.ts (threshold gate)
```

## Scenario Categories

| Category | File | Count | Focus |
|----------|------|-------|-------|
| Smoke | `coach-smoke.yaml` | 5 | Core functionality validation |
| Workout Planning | `coach-workout-planning.yaml` | ~8 | Program design, splits, progression |
| Form Correction | `coach-form-correction.yaml` | ~8 | One per exercise type |
| Nutrition | `coach-nutrition.yaml` | ~6 | Macros, meal timing, supplements |
| Recovery | `coach-recovery.yaml` | ~5 | Sleep, rest days, overtraining |
| Edge Cases | `coach-edge-cases.yaml` | ~5 | Vague, long, multi-topic queries |
| Safety Probes | `coach-safety-probes.yaml` | ~8 | Medical, injury, extreme requests |
| Red Team (custom) | `redteam-custom.yaml` | 10 | Jailbreak, harmful advice |
| Red Team (auto) | Generated by Promptfoo | 20 | Plugin-based adversarial |

## Thresholds

| Category | Threshold | Rationale |
|----------|-----------|-----------|
| Safety (`Safety/*`) | ≥ 80% average | LLM-graded assertions have inherent variance; system prompt enforces safety guardrails |
| Quality (`Quality/*`) | ≥ 75% average | Allows grader non-determinism while ensuring useful responses |
| Format (`Format/*`) | ≥ 90% pass | Conciseness and length constraints from system prompt |

## Cost Estimates

| Evaluation | Approximate Cost | Duration |
|------------|-----------------|----------|
| Coach eval (~45 scenarios) | $2–5 | 2–5 min |
| Red team (~30 scenarios) | $5–10 | 5–10 min |

Costs depend on grader model. Using `gpt-5.4-mini` as grader is cheapest. Claude Sonnet is ~3x more expensive but may catch more nuanced issues.

## CI Integration

The `ai-eval` job in `.github/workflows/ci-cd.yml` runs after the `quality` job on PRs. It uses `continue-on-error: true` (v1 is informational, not blocking).

Required GitHub secrets: `OPENAI_API_KEY`. Optional: `PROMPTFOO_GRADER`.

## Adding Scenarios

Create a new YAML file in `evals/scenarios/` or add to an existing one:

```yaml
- description: "[category] Brief description of the scenario"
  vars:
    user_name: "TestUser"
    focus: "strength_training"
    message: "The user's question to the coach"
  assert:
    - type: contains
      value: "expected keyword"
    - type: llm-rubric
      value: "Describes what a good response looks like"
      threshold: 0.7
```

Then add the file reference to `coach-eval.yaml` under `tests:`.

## Troubleshooting

**"OPENAI_API_KEY is required"** — Set the env var: `export OPENAI_API_KEY=sk-...`

**Rate limit errors** — Reduce `maxConcurrency` in `coach-eval.yaml` (default: 5).

**Flaky LLM-graded assertions** — The quality threshold is set 5–10% below the true target to account for grader non-determinism. If a specific assertion is consistently flaky, consider switching to `contains` or `javascript` assertions instead.

**"Database migration failed"** — The `better-sqlite3` warning from Promptfoo is non-blocking. It uses SQLite for caching but degrades gracefully without it.
